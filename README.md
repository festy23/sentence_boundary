# Sentence Boundary Detection для русского текста

Система определения границ предложений в русскоязычных текстах (сочинения ЕГЭ). Сравнение нескольких подходов: правила, CatBoost, трансформеры.

## Задача

На вход подаётся сплошной текст, на выход — позиции (символьные индексы) всех границ между предложениями. Основная сложность — диалоги, тире, кавычки и сокращения, которые нарушают простое правило «знак препинания + пробел + заглавная буква».

## Датасет

- **Источник:** 716 сочинений с сайта sdamgia.ru (задание 27 ЕГЭ)
- **Итоговый размер:** 534 текста, 20 875 предложений
- **Формат:** `data/processed/sentences.jsonl`

Каждая запись содержит `clean_text` (текст без маркеров) и `sentences` (список с позициями начала/конца каждого предложения).

## Ключевые выводы

- **CatBoost** показывает лучшее качество (F1=0.984) при минимальных ресурсах — рекомендуется для прода
- **rubert-tiny2** быстрый (~17 мс на текст, CPU) и хорошо обобщается, но уступает CatBoost
- Главная сложность — 10% границ в диалогах (тире, кавычки), которые не покрываются простыми правилами
- ~8% точек не являются границами предложений (сокращения, инициалы, многоточия)
